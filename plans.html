Multi-Language Code Compilation & Expansion

High Priority

Expand the current single-language question bank structure (which is locked to Java class setups) to fully support Python, JavaScript, TypeScript, Go, and C++. The Monaco editor and live parser must detect the user's chosen language and serve the corresponding starter skeleton.

Implementation Specs
  • Modify server/src/data/questions.ts to support dynamic schemas per language (e.g. starterCode: Record)
  • Add client-side language switcher drop-down in the interview header workspace
  • Wire active language state into the LiveKit data channel channel sync mechanism for Python agent reflection

Complex DSA Visualizer Registry

High Priority

Construct dedicated frontend visualization modules for trees, stacks, matrices, and graphs. Candidates can view dynamic updates (e.g., node swaps, traversal order, array slicing) reacting to their code state directly in the problem panel.

Implementation Specs
  • Create StackVisualizer.tsx mapping to "valid-parentheses" problem to show pushes/pops
  • Create TreeVisualizer.tsx utilizing canvas or SVG nodes for BST and Binary Tree problems
  • Extend registry.tsx dynamically to lazy-load visualizations based on active question def ID

NextAuth & JWT Authentication Integration

High Priority

Enable secure sign-up/login to persist and map interview sessions to specific user profiles. Support GitHub and Google OAuth providers for developers, alongside secure token storage.

Implementation Specs
  • Integrate Auth.js / NextAuth.js middleware inside client/app folder
  • Implement secure API route protection on Express server (`server/src/routes`) verifying JWTs
  • Map user schemas in MongoDB (`server/src/models/User.ts`) linking to `Session` IDs

Interactions History & Performance Dashboard

Medium Priority

Develop a dashboard landing page after user login to review past performances. Candidates can view historic radar chart improvements, transcripts, full evaluation logs, and progress indicators.

Implementation Specs
  • Create dashboard page `/dashboard` in Next.js client listing previous sessions
  • Add Recharts trend lines visualizing scores in Logic, Strategy, and Coding over time
  • Enable raw session report downloading in PDF or JSON format
Phase 3

Infrastructure & Concurrent Scalability

Redis Caching
Load Balancing

Concurrent Room Orchestration & UUID Sessions

High Priority

Upgrade Express server and LiveKit connection managers to support simultaneous user sessions. Ensure that voice agents are mapped to dynamic UUID keys rather than single rooms, stopping agent overlap.

Implementation Specs
  • Refactor token generation routes to bind room names explicitly to `sessionUuid`
  • Add dynamic voice agent instance spawning with unique identifiers mapping to MongoDB documents
  • Implement idle session garbage collection to shut down inactive LiveKit rooms

Redis-Backed Transient Caching

Low Priority

Cache active draft code, voice states (speaking/listening), and transient live transcript steps in a high-speed Redis cluster. Avoid high database write overheads during typing debounces.

Implementation Specs
  • Deploy local/cloud Redis instance configuration in `server/src/lib/redis.ts`
  • Buffer code keystrokes in Redis and periodically flush (e.g. every 5 seconds or upon submission) to MongoDB
  • Implement quick session recovery retrieving last state from cache after a network drop

Difficulty Levels & Agent Personality Tuning

Medium Priority

Configure the AI interviewer's behavioral profile before joining. Candidates can choose between: "Socratic / Encouraging" (guiding questions), "Strict / Silent" (minimal talk, deep reviews), and "Pressure Cooker" (interjects with sudden complexity extensions).

Implementation Specs
  • Update client landing page with pre-interview room setup forms (difficulty/tone selections)
  • Inject chosen preferences into metadata tokens generated for the LiveKit connection handshakes
  • Refactor agent.py `build_interview_instructions()` to parse tone options and alter instructions

Custom Question Sandbox & Upload

Medium Priority

Provide an administration interface where developers or premium candidates can upload custom question descriptions, starter skeletons, constraints, and test suites, immediately making them interview-ready.

Implementation Specs
  • Build form components in client (`/questions/new`) capturing structured problem fields
  • Implement backend REST endpoint (`POST /api/questions`) saving data to a new MongoDB schema
  • Modify session setup router to pull from combined static and custom question pools
